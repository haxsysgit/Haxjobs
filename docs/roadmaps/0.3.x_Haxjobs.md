# HaxJobs 0.3.x Roadmap — Dashboard and Core UI

## Major theme

0.3.x makes HaxJobs usable as a real job-search command center.

The backend/data model may exist by now, but this phase is where the user finally gets a clear interface for the daily workflow.

## Final outcome for 0.3.x

By the end of 0.3.x, a user should be able to:

1. Open HaxJobs and see what needs attention.
2. View saved jobs.
3. Track applications by status.
4. Open a job detail page.
5. See fit score, sponsorship risk, pack files, task state, and next action.
6. Trigger core Hermes tasks from the UI.
7. Understand what Hermes has done through timeline/task views.

## Product scope

Included:

- dashboard
- job inbox
- application pipeline
- job detail page
- task queue UI
- timeline display
- basic UX polish

Not included yet:

- Chrome extension
- Firefox extension
- assisted apply automation
- outreach sending
- deep profile survey flow

# Risks and decisions

## Risk: UI exposes too much internal data

The UI should show useful summaries first. Raw JSON can exist in debug drawers, not as the default experience.

## Risk: over-polishing before the loop works

Keep this phase practical. Beautiful enough, not perfect.

## Exit criteria for 0.3.x

0.3.x is complete when a user can manage saved jobs/applications visually and see Hermes task status/results from the UI.
