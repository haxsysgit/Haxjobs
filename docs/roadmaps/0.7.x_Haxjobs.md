# HaxJobs 0.7.x Roadmap — Assisted Apply and Outreach

## Major theme

0.7.x adds the risky but powerful workflows: application assistance and recruiter/contact outreach.

This comes after profile, packs, tasks, and approvals because this is where mistakes can cause real-world damage.

The rule is simple:

```text
Hermes can assist.
User approves high-stakes actions.
HaxJobs records what happened.
```

## Final outcome for 0.7.x

By the end of 0.7.x, HaxJobs should support:

1. Approval checkpoints.
2. Apply-assist state tracking.
3. User-input/blocker UI.
4. Contact discovery review.
5. Outreach draft review.
6. Message approval flow.
7. Full audit timeline for assisted actions.

## Product scope

Included:

- approval checkpoints
- apply-assist UI
- blockers/questions UI
- contact records UI
- outreach draft UI
- message approval flow
- audit trail

Not included:

- mass auto-apply
- mass messaging
- raw password storage
- unapproved final submits
- unapproved outreach sending

# 0.7.0 — Approval checkpoints

## Goal

Create first-class approval records for risky actions.

## Approval action types

- submit_application
- send_message
- use_sensitive_answer
- confirm_legal_answer
- upload_final_document if needed

## Build

- ApprovalCheckpoint API
- approval UI component
- approval summary text
- approved_by
- approved_at
- optional expires_at
- timeline events for approval decisions

## Done when

- Hermes can check whether approval exists before a high-stakes action

# 0.7.1 — Needs user input UI

## Goal

Give Hermes a clean way to stop and ask.

## Question types

- text answer
- yes/no
- multiple choice
- sensitive confirmation
- file needed
- login/session needed

## Build

- needs_user_input task display
- questions list
- answer submission
- answer → SavedAnswer/ProfileFact mapping where appropriate

## Done when

- Hermes can pause and HaxJobs can collect the missing input

# 0.7.3 — Final submit approval flow

## Goal

Make final submission safe.

## Flow

```text
Hermes fills what it can
→ Hermes stops before final submit
→ HaxJobs shows summary
→ user approves or rejects
→ Hermes submits only if approved
→ HaxJobs records final outcome
```

## Done when

- no application can be marked submitted unless approved and actually submitted

---

- channel recommendation
- risks/notes

## Message style

- short
- natural
- specific
- honest
- not spammy

# 0.7.6 — Message sending approval flow

## Goal

Create the safe path from draft to sent.

## Flow

```text
drafted
→ user reviews/edits
→ user approves
→ Hermes/user sends
→ status updates to sent
```

## Done when

- no message can be sent without approval
- sent status is only recorded when sending actually happened

# Risks and decisions

## Risk: accidental submission/message

Approval gates must be enforced at the workflow level, not just UI copy.

## Risk: storing credentials

Do not store raw passwords. Prefer manual auth/session handoff unless secure secret storage is intentionally designed later.

## Risk: spammy outreach

Outreach should be selective and relevant, not a mass-messaging engine.

## Exit criteria for 0.7.x

0.7.x is complete when Hermes can assist with applications/outreach while HaxJobs handles approvals, blockers, and audit history safely.
