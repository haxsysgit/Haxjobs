# HaxJobs 0.6.x Roadmap — Firefox Extension

## Major theme

0.6.x brings the browser capture workflow to Firefox.

This phase should reuse as much Chrome extension work as possible. The goal is not to create a second divergent extension. The goal is a shared extension codebase with browser-specific packaging where necessary.

## Final outcome for 0.6.x

By the end of 0.6.x, a user should be able to:

1. Load the Firefox extension locally.
2. Save a job page to HaxJobs.
3. Capture the same snapshot fields as Chrome.
4. Use the same HaxJobs backend API.
5. Build Chrome and Firefox packages without copy-paste drift.

## Product scope

Included:

- Firefox-compatible manifest/config
- shared extension source layout
- browser-specific manifest generation if needed
- Firefox popup/content script behavior
- cross-browser QA

Not included yet:

- mobile browser extension support
- extension store publishing polish
- apply automation in extension

# 0.6.0 — Firefox extension port

## Goal

Make the Chrome capture workflow work in Firefox.

## Build

- Firefox manifest adjustments
- permissions review
- popup compatibility
- content script compatibility
- API request compatibility

## Done when

- Firefox loads the extension locally
- popup opens
- Save to HaxJobs works on a generic page

---

# 0.6.1 — Shared extension source layout

## Goal

Avoid maintaining two separate extensions manually.

## Build

Suggested shape:

```text
extensions/
  shared/
    popup/
    content/
    api/
    sourceDetection.js
  chrome/
    manifest.json
  firefox/
    manifest.json
  scripts/
    build-chrome.js
    build-firefox.js
```

## Done when

- shared logic is not duplicated unnecessarily
- browser-specific files are small and obvious

# 0.6.2 — Browser-specific manifest generation

## Goal

Make packaging repeatable.

## Build

- build script for Chrome
- build script for Firefox
- output directories
- basic validation step

## Done when

- developer can build both extension variants with commands

# 0.6.3 — Firefox privacy/safety parity

## Goal

Make Firefox behavior as safe as Chrome behavior.

## Verify

- explicit click-only capture
- no background browsing history capture
- no intentional password/form-field capture
- clear success/error state
- user note support

## Done when

- Firefox follows the same privacy expectations as Chrome

---

## Goal

Test Chrome and Firefox side-by-side.

## Platforms

- LinkedIn

- Reed

## Done when

- both extensions capture useful snapshots

---

# Risks and decisions

## Risk: copy-paste drift

Do not fork the extension into two unrelated codebases.

## Risk: browser-specific quirks eat time

Keep the extension simple. Snapshot capture only.

## Exit criteria for 0.6.x

0.6.x is complete when Firefox supports the same Save to HaxJobs workflow as Chrome, using shared code where practical.
