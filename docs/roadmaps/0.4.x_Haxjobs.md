# HaxJobs 0.4.x Roadmap — Profile, Survey Flow, Packs, and Refinement

## Major theme

0.4.x makes HaxJobs the interface Hermes uses to understand the user.

This phase is about truthful reusable memory:

- profile facts
- saved answers
- sensitive answer handling
- guided survey questions
- application pack library
- pack review flow

This is where HaxJobs becomes more than a tracker. It becomes the place where the user and Hermes build shared context safely.

## Final outcome for 0.4.x

By the end of 0.4.x, a user should be able to:

1. View and edit their career profile.
2. Answer guided profile/application questions from HaxJobs.
3. See which facts are confirmed vs inferred/weak.
4. Save reusable answers.
5. Mark sensitive answers correctly.
6. Request a generated pack from Hermes.
7. View generated pack files.
8. Reuse previous packs and answers safely.

## Product scope

Included:

- profile page
- guided survey UI
- ProfileFact management
- SavedAnswer management
- sensitivity labels
- pack library
- pack generation flow
- pack/document review
- profile refinement UX

Not included yet:

- browser extension
- apply automation
- outreach sending
- mass profile import from every source

# 0.4.2 — Sensitive answer handling

## Goal

Prevent unsafe reuse of risky answers.

## Sensitivity categories

- normal
- review_before_use
- legal_sensitive
- never_auto_answer

## Sensitive topics

- visa/work authorization
- sponsorship
- salary numbers
- demographic disclosures
- disability disclosures
- criminal/legal declarations
- certification checkboxes
- anything requiring “I confirm this is true”

## Behavior

- Hermes can reuse normal confirmed answers.
- Hermes should ask before using review_before_use answers.
- Hermes must stop for legal_sensitive answers if stale/unconfirmed.
- Hermes should never auto-answer never_auto_answer fields.

## Done when

- saved answers carry sensitivity labels
- UI makes sensitive answers obvious
- approval/confirmation need is visible

# Risks and decisions

## Risk: storing fake certainty

Never promote inferred facts to confirmed without user confirmation.

## Risk: profile becomes a dumping ground

Keep facts structured and evidence-backed.

## Risk: packs hide unsafe claims

Pack outputs should be reviewable, and submitted documents must not include internal warnings/fit notes.

## Exit criteria for 0.4.x

0.4.x is complete when HaxJobs can collect/maintain user profile truth, let Hermes ask questions through the UI, and store/review generated application packs.
