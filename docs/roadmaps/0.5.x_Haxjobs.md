# HaxJobs 0.5.x Roadmap — Chrome Extension

## Major theme

0.5.x brings HaxJobs into the user's real browsing flow through a Chrome extension.

The extension should be boring and reliable. Its first job is not perfect parsing. Its first job is explicit capture:

```text
User clicks Save to HaxJobs
→ extension captures page context
→ HaxJobs stores JobSourceSnapshot
→ Hermes normalizes later
```

## Final outcome for 0.5.x

By the end of 0.5.x, a user should be able to:

1. Load the Chrome extension locally.
2. Visit a job page.
3. Click “Save to HaxJobs”.
4. Optionally add a note.
5. Save URL/title/visible text/selected text into HaxJobs.
6. See the saved snapshot/job in HaxJobs.
7. Ask Hermes to analyze it later.

## Product scope

Included:

- Chrome Manifest V3 extension
- popup UI
- content script
- source-platform guessing
- explicit click-only capture
- HaxJobs API config
- snapshot POST
- basic error/success states

Not included yet:

- background auto-saving
- scraping behind login walls without user action
- browser automation/apply flow
- Firefox support
- perfect platform-specific parsing

## Goal

Create a loadable Chrome extension.

## Build

- manifest.json
- popup HTML/JS/CSS or framework build
- content script
- background/service worker if needed
- extension build command
- local API URL setting

## Done when

# 0.5.1 — Capture current page snapshot

## Goal

Capture useful page context on user click.

## Capture fields

- current URL
- page title
- visible page text
- selected text if any
- source platform guess
- timestamp
- optional user note

## Done when

- popup can collect data from the active tab/content script
- payload shape matches HaxJobs JobSourceSnapshot API

---

# 0.5.5 — Chrome privacy and safety pass

## Goal

Make capture explicit and respectful.

## Rules

- only capture after explicit click
- show what is being saved where possible
- do not intentionally capture passwords
- do not auto-save browsing history
- do not silently monitor tabs
- allow deletion from HaxJobs side

## Done when

- extension behavior is obvious and user-controlled

# Risks and decisions

## Risk: extension tries to parse too much

Do not make the extension responsible for perfect job extraction. Hermes normalizes later.

## Risk: privacy creep

No background capture. No auto-browsing history. Click-only.

## Exit criteria for 0.5.x

0.5.x is complete when Chrome users can save job-page snapshots into HaxJobs reliably.
