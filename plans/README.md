- Full migration from intake JSON to SQLite-only source of truth: real and important, but too large/risky for this cleanup wave. Plan 007 adds characterization tests first; a later plan should demote intake JSON to raw import/archive after those tests exist.
- First-class HermesTask and ApprovalCheckpoint tables: aligned with product docs, but larger than this wave. Plan 004 tightens existing approval transitions first; a future schema plan can introduce explicit approval records.
- Telegram-first action loop: strong product direction, but should follow API hardening and approval transition fixes to avoid accidental unsafe state changes.
