Command line options
Options and flags for the Codex terminal client

How to read this reference
This page catalogs every documented Codex CLI command and flag. The CLI inherits most defaults from ~/.codex/config.toml. Any -c key=value overrides passed at the command line take precedence for that invocation.

Global flags
--add-dir path: Grant additional directories write access alongside the main workspace. Repeatable.
--ask-for-approval, -a: untrusted | on-request | never. Control when Codex pauses for human approval before running a command. on-failure is deprecated.
--cd, -C path: Set working directory.
--config, -c key=value: Override configuration values. Values parse as TOML if possible.
--dangerously-bypass-approvals-and-sandbox, --yolo: Run every command without approvals or sandboxing. Only use inside an externally hardened environment.
--disable feature / --enable feature: toggle feature flags.
--image, -i path[,path...]: Attach images.
--model, -m string: Override the configured model, e.g. gpt-5.4.
--oss: Use local open source model provider, equivalent to -c model_provider="oss" and validates Ollama is running.
--profile, -p string: Layer $CODEX_HOME/profile-name.config.toml on top of base user config.
--remote ws://host:port | wss://host:port | unix:// | unix://PATH: Connect to remote app-server endpoint.
--remote-auth-token-env ENV_VAR: Bearer token env var for remote auth.
--sandbox, -s read-only | workspace-write | danger-full-access: Select sandbox policy.
--search: Enable live web search.
--strict-config: Error when config.toml contains unrecognized fields.
PROMPT: Optional text instruction to start session.

Command overview
codex: Stable. Launch TUI.
codex app-server: Experimental. Launch Codex app server for local development/debugging over stdio, WebSocket, or Unix socket.
codex debug models: Experimental. Print raw model catalog Codex sees.
codex doctor: Stable. Diagnostic report.
codex exec: Stable. Run Codex non-interactively. Alias codex e. Stream results to stdout or JSONL and optionally resume previous sessions.
codex login: Stable. Authenticate using ChatGPT OAuth, device auth, API key, or access token from stdin.
codex mcp: Experimental. Manage MCP servers.
codex mcp-server: Experimental. Run Codex itself as an MCP server over stdio.
codex resume/fork: Stable. Continue or fork sessions.

codex app-server
--listen stdio:// | ws://IP:PORT | unix:// | unix://PATH | off: transport listener URL.
--stdio: equivalent to --listen stdio://.
--ws-auth capability-token | signed-bearer-token: WebSocket auth.
--ws-token-file / --ws-token-sha256: capability token config.
--ws-shared-secret-file: HMAC secret for JWT bearer token validation.

codex debug models
--bundled: skip refresh and print only bundled model catalog.

codex exec
Use codex exec (or codex e) for scripted/CI-style runs that should finish without human interaction.
Flags include:
--cd, -C path
--color always | never | auto
--dangerously-bypass-approvals-and-sandbox, --yolo
--ephemeral
--ignore-user-config
--image, -i
--json, --experimental-json: Print newline-delimited JSON events.
--model, -m string
--oss
--output-last-message, -o path: Write assistant final message to file.
--output-schema path: JSON Schema file describing expected final response shape. Codex validates tool output against it.
--profile, -p string
--sandbox, -s read-only | workspace-write | danger-full-access
--skip-git-repo-check
-c, --config key=value
PROMPT string | - (read stdin)
Resume subcommand: codex exec resume [SESSION_ID]

Codex writes formatted output by default. Add --json for newline-delimited JSON events. Use --last to resume most recent session from current working directory; add --all to search across all sessions.

codex login
--device-auth: OAuth device code flow.
--with-access-token: read access token from stdin.
--with-api-key: read API key from stdin, e.g. printenv OPENAI_API_KEY | codex login --with-api-key.
status subcommand exits 0 when credentials are present.

codex mcp
Manage MCP server entries stored in ~/.codex/config.toml.
add <name> -- <command...> or --url <value>; supports --env KEY=VALUE, --bearer-token-env-var, OAuth fields.
list/get support --json.

codex mcp-server
Run Codex as an MCP server over stdio so other tools can connect. Inherits global config overrides and exits when downstream client closes.

Safety tips
Use --sandbox workspace-write for unattended local work that can stay inside the workspace. Avoid --dangerously-bypass-approvals-and-sandbox unless inside a dedicated sandbox VM. Prefer --add-dir over danger-full-access for extra writable roots. Pair --json with --output-last-message in CI.
